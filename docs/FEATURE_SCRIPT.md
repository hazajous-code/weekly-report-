# WeeklyFlow 기능 설명 · 시연 스크립트

발표/데모에서 그대로 읽을 수 있는 **내레이션 스크립트**입니다.
각 기능은 **[목적] · [화면] · 🎙️[멘트] · [시연 동선] · 💡[AX 가치]** 형식으로 구성했습니다.

- 라이브: https://hazajous-code.github.io/weekly-report-/
- 권장 시연 계정 흐름: 팀장 김선우 → 현업 이서연 → 실장 오세훈 → 전략기획 서지안 → 임원 노상헌
- 시작 전 팁: 사이드바 하단 **♻️ 데이터 초기화**로 샘플을 새로 채운 뒤 시연하세요.

---

## 0. 오프닝 (30초)

🎙️ *"많은 조직에서 주간보고는 메일·엑셀·문서로 흩어져 있고, 팀장과 실장이 일일이 취합·복사·정리하는 데 시간을 씁니다. WeeklyFlow는 **현업이 한 번 입력하면, 팀장 → 실장 → 전략기획 → 임원까지 자동으로 취합·요약되는 단일 포털**입니다. 오늘은 보고 한 건이 임원 보고서가 되기까지의 흐름을 실제 화면으로 보여드리겠습니다."*

💡 핵심 메시지 3가지: **① 취합 시간 절감 ② 누락·중복 방지 ③ 임원 보고 자동화.**

---

## 1. 로그인 · 역할 선택

[목적] 실제 인증 없이 5개 역할의 화면·권한 차이를 즉시 체험.
[화면] 역할 카드 5종(현업/팀장/실장/전략기획/임원) → 계정 선택.

🎙️ *"WeeklyFlow는 역할 기반 포털입니다. 같은 시스템이지만 현업·팀장·실장·전략기획·임원이 각각 다른 화면과 권한을 봅니다. 데모에서는 역할을 골라 바로 들어가 보겠습니다. 실제 운영에서는 이 부분이 **사내 SSO와 조직도 연동**으로 대체되어, 로그인하면 본인의 소속·역할이 자동으로 결정됩니다."*

[시연 동선] '팀장' 카드 클릭 → '김선우 · 포털기획팀' 로그인.
💡 역할별 접근 제어가 메뉴·데이터 범위에 그대로 반영됨(현업은 본인 것만, 팀장은 팀, 실장은 실, 전략·임원은 전사).

---

## 2. 현업 — 주간보고 작성 + AI 미리보기

[목적] 진행 내용·정량/정성 성과·이슈·계획·지원요청을 구조화해 입력.
[화면] 보고 작성 폼 + 우측 'AI 요약 미리보기' 패널. 임시저장/제출.

🎙️ *"현업 담당자는 한 주의 업무를 정해진 항목에 맞춰 입력합니다. 진행 내용, **정량 성과(수치)와 정성 성과(질적 변화)를 구분**해 적게 해서 보고 품질을 표준화했습니다. 작성 중 'AI로 요약 생성'을 누르면, 입력 내용을 **핵심 성과·주요 이슈·다음 주 중점·임원 확인 필요** 네 가지로 정리해 미리 보여줍니다. 제출 전 스스로 점검할 수 있죠."*

[시연 동선] (현업 이서연 로그인) → 반려된 보고 열기 → 'AI로 요약 생성' 클릭 → 4개 섹션 확인 → 보완 후 '제출'.
💡 입력 표준화 + 자기 점검으로, 뒤 단계의 재작업·반려를 줄임.

---

## 3. 팀장 — 1차 검토(승인/반려/피드백)

[목적] 팀원 보고를 가장 먼저 검토하는 **중간 결재 단계**.
[화면] 팀 검토 대시보드(제출률·검토대기·승인·반려) → 보고서 상세에서 승인/반려/코멘트.

🎙️ *"팀장 화면입니다. 상단에 **팀 제출률과 검토 대기 건수**가 한눈에 보입니다. 어떤 팀원이 아직 안 냈는지, 무엇을 검토해야 하는지 바로 파악됩니다. 보고서를 열면 승인·반려·코멘트를 남길 수 있고, **반려 사유는 필수**로 받아 피드백이 누락되지 않게 했습니다. 반려하면 현업에게 사유와 함께 돌아가고, 보완 후 재제출됩니다."*

[시연 동선] (팀장) 대시보드 → 미검토 보고 행 클릭 → 검토 의견 입력 → '승인' 또는 '반려'.
💡 '중간 결재'가 있어 실장에게 정제된 내용만 올라감 → 상위 단계 부담 감소.

---

## 4. 팀장 — 팀 요약 작성 (AI 일괄 요약)

[목적] 승인된 보고들을 묶어 **팀 단위 요약** 생성 → 실장에 제출.
[화면] 승인 보고 목록 + 'AI 자동 요약' + 헤드라인/핵심성과/이슈 편집.

🎙️ *"승인된 보고가 모이면, '승인 보고 일괄 요약' 버튼 하나로 **팀 요약 초안이 자동 생성**됩니다. 팀장은 문구만 다듬어 실장에게 제출하면 됩니다. 예전에는 팀장이 팀원 보고를 복사·붙여넣기로 정리했지만, 여기서는 그 과정이 자동화됩니다."*

[시연 동선] '팀 요약 작성' 메뉴 → 'AI 일괄 요약' 클릭 → 자동 채워진 내용 확인 → '실장에게 제출'.
💡 팀장의 **취합 노동 제거** — 이 시스템의 첫 번째 시간 절감 포인트.

---

## 5. 실장 — 팀 요약 검토 + 실(室) 요약

[목적] 팀장이 올린 팀 요약을 **2차 검토**하고, 실 단위로 한 번 더 취합.
[화면] 실 검토 대시보드(팀별 제출 현황) → 팀 요약 상세 승인/반려 → 실 요약 작성.

🎙️ *"실장은 산하 팀들이 올린 팀 요약을 한 화면에서 검토합니다. 승인/반려가 가능하고, 승인된 팀 요약을 묶어 '팀 요약 일괄 취합'으로 **실 요약 초안**을 만듭니다. 여기에 **임원 의사결정이 필요한 사항**을 따로 적어 전략기획팀에 올립니다. 보고가 한 계단씩 올라갈수록 정보가 압축되는 구조입니다."*

[시연 동선] (실장 오세훈) 대시보드 → 팀 요약 행 클릭 → 승인 → '실 요약 작성' → 'AI 취합' → '전략기획팀에 제출'.
💡 **계층적 압축**(보고→팀→실)으로 상위로 갈수록 핵심만 남음 → 의사결정 속도 향상.

---

## 6. 전략기획팀 — 전사 취합 대시보드 + 자동 분류

[목적] 전사 제출 현황을 관제하고, 실별 요약을 **성과·이슈·의사결정**으로 자동 분류.
[화면] 전체 제출률·실 요약 제출 수·의사결정 필요·지연 제출자 + 실별 현황 + 자동 분류 카드.

🎙️ *"전략기획팀은 전사 관제탑입니다. **전체 제출률, 실별 제출 현황, 지연 제출자**가 한눈에 보이고, 마감 미제출자는 자동으로 식별됩니다. 오른쪽에는 각 실의 요약이 **핵심 성과 / 주요 이슈 / 임원 의사결정 필요**로 자동 분류되어 표시됩니다. 어느 부서에서 무슨 이슈가 올라왔는지 일일이 문서를 열지 않아도 됩니다."*

[시연 동선] (전략기획 서지안) 취합 대시보드에서 지표·지연자·자동 분류 카드 설명.
💡 **누락·지연 가시화** + 자동 분류로 전사 현황 파악 시간을 분 단위로 단축.

---

## 7. 전략기획팀 — 임원 보고용 요약 AI 생성

[목적] 실별 요약을 취합해 임원 보고 4개 섹션을 자동 생성·편집·제출.
[화면] 'AI로 임원 보고 초안 생성' → 핵심성과/주요이슈/리스크/의사결정 편집 → 제출.

🎙️ *"버튼 하나로 모든 실의 요약을 모아 **임원 보고 초안**을 만듭니다. 핵심 성과, 주요 이슈, 리스크, 의사결정 필요 사항 네 가지로 정리되며, 담당자는 검토·보정 후 제출합니다. 매주 반복되던 '임원 보고서 만들기'가 **초안 자동화 + 사람 검수**로 바뀝니다."*

[시연 동선] '임원 보고 요약 생성' 메뉴 → 'AI 초안 생성' → 내용 확인 → '임원 보고 제출'.
💡 이 시스템의 **하이라이트** — 임원 보고서 작성 자동화. (실제 운영 시 PDF/PPT 자동 산출로 확장)

---

## 8. 임원 — 최종 검토 + 의사결정 코멘트

[목적] 취합된 보고를 빠르게 확인하고 지시·피드백을 남김.
[화면] 핵심성과/이슈/리스크/의사결정 4개 패널 + 실별 주요 성과 + 코멘트.

🎙️ *"임원 화면은 **읽는 데 최적화**되어 있습니다. 한 화면에서 핵심 성과·이슈·리스크·의사결정 필요 사항을 보고, 실별 주요 성과도 표로 확인합니다. 코멘트를 남기면 전략기획팀으로 회신되어, **의사결정이 다시 현장으로 흐릅니다.** 보고가 일방향으로 끝나지 않고 양방향 소통이 되는 구조입니다."*

[시연 동선] (임원 노상헌) 4개 패널 훑기 → 코멘트 입력 → '코멘트 등록'.
💡 임원의 **확인·의사결정 시간 단축** + 피드백 루프 완성.

---

## 9. 피드백 · 이력 (감사 추적)

[목적] 보고서별로 누가·언제·무엇을(제출/승인/반려/코멘트) 했는지 타임라인으로 추적.
[화면] 보고서 목록 + 피드백 타임라인.

🎙️ *"모든 보고는 **변경·결재 이력이 타임라인으로 남습니다.** 누가 언제 제출했고, 누가 어떤 사유로 반려했는지 투명하게 보입니다. 커뮤니케이션 누락을 막고, 분쟁 시 근거가 됩니다. 실제 운영에서는 이 부분이 **감사 로그**로 확장됩니다."*

[시연 동선] '피드백·이력' 메뉴 → 보고서 선택 → 타임라인 확인.
💡 **투명성·추적성** — 거버넌스와 신뢰의 기반.

---

## 10. 상태 관리 · 대시보드 지표

🎙️ *"모든 보고와 요약은 작성 중 → 제출 → 승인/반려 상태로 관리되고, **상태 배지**로 한눈에 구분됩니다. 각 역할의 대시보드에는 제출률, 검토 대기, 승인/반려, 지연 제출자 같은 지표가 실시간으로 집계됩니다. 관리자는 '지금 우리 조직 보고가 어디까지 왔는지'를 숫자로 봅니다."*

💡 정성적 보고를 **정량 지표**로 전환 — 운영 가시성 확보.

---

## 11. 클로징 (전사 확장 메시지)

🎙️ *"방금 보신 것처럼, 보고 한 건이 작성→팀장→실장→전략기획→임원까지 **사람이 복사·정리하지 않아도** 자동으로 취합·요약됩니다. 이 프로토타입은 흐름과 가치를 증명한 것이고, **전사 포털로 가려면 백엔드·사내 SSO·조직도 연동·설정 가능한 결재선·보안 심의**가 더해지면 됩니다. 그 위에 실제 AI 요약과 BI 분석을 올리면, '보고 취합 시간 X시간 절감' 같은 **AX 성과를 수치로 증명**할 수 있습니다. 한 본부 파일럿으로 시작해 확산하는 것을 제안드립니다."*

### 부록 — 시연 체크리스트
- [ ] 데이터 초기화로 샘플 리셋
- [ ] 팀장: 이서연 보고 반려 → 현업: 보완 재제출 → 팀장: 승인 (반려→재제출 루프 시연)
- [ ] 팀장: 팀 요약 AI 생성 → 제출

- [ ] 임원: 코멘트 등록
- [ ] 각 단계 '상태 배지'와 '피드백 타임라인' 강조
